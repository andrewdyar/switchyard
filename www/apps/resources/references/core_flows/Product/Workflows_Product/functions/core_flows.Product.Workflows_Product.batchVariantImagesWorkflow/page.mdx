# batchVariantImagesWorkflow - Switchyard Core Workflows Reference

This documentation provides a reference to the `batchVariantImagesWorkflow`. It belongs to the `@switchyard/medusa/core-flows` package.

This workflow manages the association between product variants and images in bulk.
It's used by the [Batch Variant Images Admin API Route](https://docs.switchyard.com/api/admin#products\_postproductsidvariantsvariant\_idimagesbatch).

You can use this workflow within your own customizations or custom workflows to manage variant-image associations in bulk.
This is also useful when writing a [seed script](https://docs.switchyard.com/learn/fundamentals/custom-cli-scripts/seed-data) or a custom import script.

:::note

This is available starting from [Switchyard v2.11.2](https://github.com/switchyard/medusa/releases/tag/v2.11.2).

:::

import type {
  SwitchyardRequest,
  SwitchyardResponse,
} from "@switchyard/framework/http"
import { batchVariantImagesWorkflow } from "@switchyard/medusa/core-flows"

export async function POST(
  req: SwitchyardRequest,
  res: SwitchyardResponse
) {
  const { result } = await batchVariantImagesWorkflow(req.scope)
    .run({
      input: {
        variant_id: "variant_123",
        add: ["img_123", "img_456"],
        remove: ["img_789"]
      }
    })

  res.send(result)
}
